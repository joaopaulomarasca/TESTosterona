Dando Estilo à Página — Introdução ao CSS

❓ O que é o CSS e como ele funciona?

O CSS funciona como um conjunto de regras que dizem ao navegador como apresentar os elementos HTML. Ele controla cores, tamanhos, espaçamentos, fontes, posicionamento e muitos outros aspectos visuais.

Uma regra CSS é formada por duas partes principais:

  • Seletor: aponta para qual(is) elemento(s) HTML a regra deve ser aplicada. Pode ser uma tag (p, h1), uma classe (.minha-classe) ou um ID (#meu-id).
  • Declaração: define o estilo a ser aplicado, formada por uma propriedade e um valor, separados por dois pontos (:).
sintaxe básica
p {
  color: blue;
  font-size: 16px;
}

/* seletor { propriedade: valor; } */

Este parágrafo ficou azul.

Este também, pois o seletor p pega todos.

Formas de Aplicar CSS

Existem três maneiras principais de conectar o CSS ao seu HTML:

CSS Inline

Aplicado diretamente na tag HTML usando o atributo style. Deve ser evitado em projetos, pois mistura estrutura com estilo e dificulta a manutenção.

css inline
<p style="color: red; font-weight: bold;">
  Este texto é vermelho e negrito.
</p>

Este texto é vermelho e negrito.

CSS Interno

Feito usando a tag <style> dentro do <head>. Útil para estilizar um único documento HTML, mas não recomendado para sites com múltiplas páginas.

html
<head>
  <style>
    h1 {
      color: red;
    }
  </style>
</head>

CSS Externo (Recomendado)

Reside em um arquivo separado com extensão .css. Para vincular ao HTML, adiciona-se um elemento <link> dentro do <head>. É o método mais usado no desenvolvimento web.

css externo
/* No arquivo index.html */
<head>
  <link rel="stylesheet"
        href="style.css">
</head>

/* No arquivo style.css */
body {
  background-color: #f0f0f0;
  font-family: Arial, sans-serif;
}

O fundo e a fonte da página seriam alterados globalmente pelo arquivo style.css vinculado.

Exemplo Prático: Estilizando nossa primeira página

css — style.css
body {
  font-family: Arial, sans-serif;
  background-color: #f4f4f4;
  margin: 20px;
}

h1 {
  color: #333;
  text-align: center;
}

p {
  line-height: 1.6;
  color: #666;
}

Seletores e a Cascata

🫵 Seletores: Onde o CSS Encontra o HTML

Um seletor é o que "seleciona" os elementos HTML que você quer estilizar. A escolha do seletor certo é fundamental para a organização do seu código.

Seletor de Tipo (ou de Elemento)

Seleciona todos os elementos de uma determinada tag.

seletor de tipo
p {
  color: gray;
}

/* Afeta TODOS os <p> da página */

Primeiro parágrafo — cinza.

Segundo parágrafo — também cinza.

Todos os <p> são afetados.

Seletor de Classe

O tipo de seletor mais comum. Você adiciona o atributo class no HTML e o seleciona no CSS usando um ponto (.). Classes podem ser reutilizadas em quantos elementos você quiser.

seletor de classe
/* HTML */
<h2 class="titulo-destaque">Título</h2>
<p class="titulo-destaque">Parágrafo</p>

/* CSS */
.titulo-destaque {
  font-weight: bold;
  color: purple;
}

Título importante

Este parágrafo também é destaque

Seletor de ID

Usado para selecionar um único elemento específico na página. No CSS, o seletor usa o sinal de hash (#). Uma regra crucial: cada id deve ser único em toda a página.

seletor de id
/* HTML */
<div id="cabecalho-principal">
  <h1>Meu Website</h1>
</div>

/* CSS */
#cabecalho-principal {
  background-color: #333;
  color: white;
  padding: 20px;
}

Meu Website

Entendendo a Cascata e a Especificidade

A palavra "cascata" no nome do CSS não está ali por acaso. A Cascata é a ordem de precedência que o navegador usa para decidir qual regra aplicar quando várias estão "competindo" pelo mesmo elemento.

O seletor com maior especificidade vence. Pense na especificidade como um sistema de pontuação — a regra com a pontuação mais alta é aplicada.

1
!importantQuase sempre vence — use com muito cuidado 💩
2
Estilos Inlinestyle="..." direto no elemento
3
Seletores de ID#meu-id
4
Classes, Atributos e Pseudo-classes.classe · [attr] · :hover
5
Tags e Pseudo-elementosp · h1 · ::before
6
Ordem no códigoA última regra declarada vence em caso de empate

Exemplo de Especificidade

qual cor vence?
/* HTML */
<p id="paragrafo-id"
   class="paragrafo-classe">
  Qual cor sou eu?
</p>

/* CSS */
p { color: green; } /* mais fraco */
.paragrafo-classe { color: blue; }
#paragrafo-id { color: red; } /* vence! */

→ O texto ficará vermelho

O ID tem maior especificidade e vence as regras de tipo e classe.

💡

Quando a especificidade é a mesma (ex: duas classes), a regra que aparece por último no código CSS é a que vence e é aplicada.

Exemplo Prático: Combinando Seletores

css — estilos.css
/* Seletor de Tipo */
p {
  font-family: sans-serif;
  line-height: 1.5;
}

/* Seletor de Classe */
.destaque {
  font-weight: bold;
  color: darkred;
}

/* Seletor de ID */
#cabecalho {
  background-color: #333;
  color: white;
  text-align: center;
  padding: 15px;
}

Seletores Avançados

Os seletores avançados permitem mirar nos elementos não apenas pelo nome, classe ou ID, mas também pelo relacionamento hierárquico ou pela combinação de suas propriedades.

1. Seletor de Agrupamento

Permite aplicar o mesmo conjunto de regras a vários seletores, evitando repetição de código (princípio DRY — Don't Repeat Yourself).

agrupamento com vírgula
/* Aplica à todos de uma vez */
h1, h2, .aviso {
  color: red;
  text-align: center;
}

2. Seletor Descendente

Aplica estilo a um elemento que está dentro de outro, não importando a profundidade. Usa um espaço entre os seletores.

descendente — espaço entre seletores
/* Todos os <a> dentro de <nav> */
nav a {
  color: green;
}

<nav>
  <p>
    <a>Link da Nav</a> ✅ selecionado
  </p>
</nav>
<a>Link fora</a> ❌ não selecionado
<nav> Link da Nav ✅

3. Seletor Filho Direto

Aplica estilo a um elemento apenas se ele for filho imediato do pai. Usa o sinal de maior que (>).

filho direto — sinal >
/* Apenas os <li> filhos diretos de <ul> */
ul > li {
  font-size: 20px;
}

<ul>
  <li>Item 1</li>    ✅ filho direto
  <li>
    <ul>
      <li>Subitem</li> ❌ neto

4. Seletor de Encadeamento

Aplica estilo a um elemento que corresponde a todos os seletores ao mesmo tempo. Sem espaço entre os seletores.

encadeamento — sem espaço
/* Apenas <p> que tem AS DUAS classes */
p.destaque.alerta {
  border: 1px solid black;
  background: lightyellow;
}

<p class="destaque alerta">
  Selecionado ✅
</p>
<p class="destaque">
  Não selecionado ❌
</p>

💻 O Segredo do Layout: A Propriedade display

Se o HTML define os elementos da sua página, o display é a propriedade CSS que define como esses elementos se comportam e interagem com o espaço ao redor deles.

Pense no display como as regras de etiqueta de um elemento: ele decide se o elemento quer sentar sozinho na mesa ou se pode compartilhar o espaço com os vizinhos.

1. display: block 🧱

O elemento "dominador" da linha. Sempre ocupa 100% da largura disponível e força o elemento seguinte a quebrar a linha. Tags como <h1>, <p> e <div> são block por padrão. Permite definir width e height livremente.

display: block — cada elemento ocupa a linha inteira
.bloco {
  display: block;
  background: #e74c3c;
  color: white;
  width: 200px;
  height: 40px;
  margin-bottom: 8px;
}
Caixa Bloco 1
Caixa Bloco 2
Caixa Bloco 3

2. display: inline 🏃

O elemento "social". Ocupa apenas o espaço necessário para o conteúdo e se enfileira ao lado dos outros. Tags como <span>, <a> e <strong> são inline por padrão. Não aceita width nem height — o tamanho é determinado só pelo conteúdo.

display: inline — elementos ficam lado a lado
.inline {
  display: inline;
  background: #3498db;
  color: white;
  padding: 4px 8px;

  /* Estas são ignoradas! */
  width: 200px;
  height: 50px;
}
Inline A Inline B maior Inline C

3. display: inline-block 👑

O melhor dos dois mundos. Fica lado a lado com outros elementos (como inline), mas ainda permite definir width e height (como block). Perfeito para menus e galerias.

display: inline-block — lado a lado com medidas definidas
.inline-block {
  display: inline-block;
  background: #2ecc71;
  color: white;
  width: 120px;  ✅ funciona!
  height: 70px;  ✅ funciona!
  text-align: center;
  line-height: 70px;
}

4. display: none 👻

O elemento desaparece completamente da tela e do fluxo do documento — não ocupa espaço, como se nunca tivesse existido. Diferente de visibility: hidden, que esconde mas mantém o espaço.

⚠️

display: none remove o elemento visualmente e do fluxo. visibility: hidden apenas o torna invisível, mas o espaço permanece ocupado.

Posicionamento CSS

O Posicionamento CSS controla como um elemento se comporta e onde ele é colocado na tela em relação ao fluxo normal do HTML, a outros elementos, ou à janela do navegador.

O segredo está em dominar os quatro valores principais da propriedade position: static, relative, absolute e fixed.

1. position: static — Padrão

É o comportamento padrão de todo elemento HTML. O elemento flui naturalmente de acordo com a ordem do HTML. As propriedades top, left, right e bottom não têm efeito aqui.

2. position: relative

Permite mover o elemento a partir de sua posição original. O espaço que ele ocupava é mantido no fluxo, mesmo que o elemento seja deslocado. Muito usado para servir de referência para elementos filhos com position: absolute.

position: relative — deslocado da posição original
.caixa-b {
  position: relative;
  top: 20px;
  left: 30px;
}

/* O espaço original ainda existe
   no fluxo do documento */

Caixa B

3. position: absolute

Retira o elemento do fluxo normal. Ele se posiciona em relação ao ancestral posicionado mais próximo (com qualquer position diferente de static). Se não houver ancestral posicionado, se posiciona em relação ao <body>.

position: absolute — dentro de um container relative
.container {
  position: relative; ← referência
  width: 200px;
  height: 120px;
  background: #f0f0f0;
}

.selo {
  position: absolute;
  top: 8px;
  right: 8px;
  background: red;
  color: white;
  padding: 4px 8px;
  border-radius: 4px;
}

Z-Index e Contexto de Empilhamento

Ao usar absolute ou fixed, o elemento entra em uma nova camada. A propriedade z-index controla a ordem de empilhamento — maior valor = mais na frente. Só funciona em elementos com position diferente de static.

z-index — quem fica na frente?
.azul {
  position: absolute;
  z-index: 1;
}
.vermelho {
  position: absolute;
  z-index: 2; ← fica na frente
}
z-index: 1
z-index: 2

4. position: fixed

O mais extremo dos posicionamentos. O elemento é posicionado em relação à janela do navegador (viewport) e permanece no mesmo lugar ao rolar a página. Perfeito para barras de navegação fixas e botões "Voltar ao topo".

💡

As propriedades de offset top, left, right e bottom só têm efeito em elementos com position diferente de static. As margens (margin) são aplicadas separadamente e não interferem no posicionamento.

🖼️ CSS Float

A propriedade float foi uma das primeiras ferramentas poderosas para organizar layout. Sua principal função é replicar o efeito de jornal ou revista, onde o texto flui elegantemente ao redor de uma imagem.

O float retira o elemento do fluxo normal do HTML, permitindo que o conteúdo vizinho suba e ocupe o espaço ao lado dele.

O Problema Padrão

Sem float, uma imagem e um parágrafo sempre ficam empilhados — o parágrafo começa em uma nova linha abaixo da imagem, mesmo que haja espaço ao lado.

Usando float: left e float: right

float: left — texto envolve a imagem pela direita
img {
  float: left;
  margin-right: 12px;
  width: 80px;
}

/* O parágrafo seguinte
   envolve a imagem pela direita */
IMG

Este é o texto do parágrafo que flui ao redor da imagem graças ao float: left. O efeito é similar a revistas e jornais. O texto continua ao lado enquanto houver espaço disponível.

float: right — imagem vai para a direita
img {
  float: right;
  margin-left: 12px;
  width: 80px;
}
IMG

Com float: right, a imagem vai para a direita e o texto envolve pelo lado esquerdo. Útil para logotipos e blocos laterais de informação.

A Propriedade clear 🧹

O float afeta todos os elementos que vêm depois dele. Quando você não quer que um elemento fique ao lado do item flutuante, usa a propriedade clear.

ValorComportamento
clear: leftNão flutua ao lado de itens com float: left

clear: bothNão flutua ao lado de itens em nenhum lado — o mais comum
clear: both — rodapé sempre abaixo dos elementos flutuantes
img {
  float: left;
}

footer {
  clear: both; ← abaixo de tudo
  background: #333;
  color: white;
  padding: 10px;
}

🚀 Flexbox: Layouts Flexíveis e Responsivos

O CSS Flexbox é uma técnica projetada para dispor, alinhar e distribuir o espaço entre elementos em um contêiner. Ele trabalha em uma dimensão por vez: ou em uma linha, ou em uma coluna.

📐

A Regra de Ouro: as propriedades do Flexbox se dividem entre as que vão no elemento pai (contêiner) e as que vão nos elementos filhos (itens).

📱 Métodos para Criar Sites Responsivos

Um site responsivo é aquele que fica ótimo e funciona bem em computadores, tablets e celulares. O CSS oferece quatro ferramentas principais para isso.

1. Media Queries 📏

Pense nas Media Queries como "regras condicionais". Elas permitem aplicar estilos apenas se uma condição for atendida — geralmente relacionada ao tamanho da tela.

media query — estilo condicional
/* Estilos padrão (desktop) */
.cabecalho {
  display: flex;
  flex-direction: row;
}

/* Quando a tela tiver 600px ou menos */
@media (max-width: 600px) {
  .cabecalho {
    flex-direction: column;
  }
}

2. CSS Flexbox ➡️

O Flexbox é o "organizador de filas" do CSS. Sistema unidimensional, excelente para alinhar itens em uma única linha ou coluna. A propriedade flex: 1 nos filhos faz com que dividam o espaço igualmente e se ajustem automaticamente.

3. CSS Grid 🖼️

O CSS Grid é o "arquiteto" do CSS. Sistema bidimensional, ideal para layouts complexos com linhas e colunas simultâneas. A unidade fr (fração) é excelente para responsividade.

css grid — layout em grade
.container {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 16px;
}

/* 1fr = 1 fração do espaço
   disponível. As 2 colunas
   dividem o espaço igualmente. */

4. Frameworks Externos (Bootstrap) 📦

Frameworks são "kits de construção prontos". O Bootstrap é o mais famoso — fornece classes CSS pré-escritas e componentes prontos (botões, navbar, carrosséis). Usa um sistema de grade de 12 colunas baseado em Flexbox.

html — bootstrap grid
<div class="row">
  <!-- ocupa 6 de 12 colunas = metade -->
  <div class="col-6">Primeiro Item</div>
  <!-- ocupa 4 de 12 colunas = um terço -->
  <div class="col-4">Segundo Item</div>
</div>

Escolhendo a Ferramenta Certa

FerramentaQuando usar
GridLayout principal da página — organização macro (cabeçalho, sidebar, conteúdo, rodapé)
FlexboxAlinhamento dentro das seções — organização micro (menus, cards, formulários)
Media QueriesAjustes pontuais quando o layout falha em telas muito pequenas ou grandes

💡

Não existe uma ferramenta "melhor". Como desenvolvedor, você vai combinar todas elas dependendo do que o projeto precisa. Grid para a estrutura, Flexbox para os detalhes, Media Queries para os ajustes.
